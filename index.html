Declarations

Let

Let has block scope if it is declared in a block and global scope if it is declared outside of a block. Also, it not accessible before it is declared. You can not redeclare the same variable in the same scope. Ex. Let me = ‘Foo’;
Let me = ‘Bar’; This would make an error.

Const

Primitives can not be changed. If you make an object with the const keyword its properties and method can still be added, subtracted, or changed. BUT you can not totally reassign the object to be a new object.

Var

Var has function scope if declared in a function and global scope if declared outside of a function.

Global Scope

Variables declared Globally (outside any function) have Global Scope. They can be used anywhere in the program.

Function Scope

Variables declared Locally (inside a function) have Function Scope. They can only be used in the function they are declared in.

Block Scope

Variables declared with the var keyword can not have Block Scope. Variables declared inside a block {} can be accessed from outside the block.

Logical Operators

&&- and (true && false) = false
||- or (true || false) = false
!- not Flips value. !true is false. !false is true.

Ternary Operator

Condition ? 1 : 2. If the condition is true use the first value if not use the second.

Short-Circuitng of Logical Operators

The || operator, for example, will return the value to its left when that can be converted to true and will return the value on its right otherwise.  Console.log(null || “Hello”); This would log Hello because null is a false value.

The && operator works similarly but the other way around. When the value to its left is something that converts to false, it returns that value, and otherwise it returns the value on its right.
“"&&"Hello" would return “” and “Something”&&"Hello" would return Hello.

In the case of true || X, no matter what Xis—even if it’s a piece of program that does something terrible—the result will be true, and X is never evaluated. The same goes for false && X, which is false and will ignore X. This is called short-circuit evaluation.

Functions

Creating Functions

Functions have the function keyword, parameters, and then a code block.

A function is created with parameters and when it is called arguments are passed into the function.

function name(x, y){
statements;
}

Once you make a function to use it you need to call it.

Functions Call

name(myX, myY);

This is how you call a function. myX and myY are arguments that are passed into the function. myX would be used inplace of x.

Function Invocation

Function invocation is when the function will be activated.

  • Event- Caused by an event.
  • Invoked- Invoked by JavaScript.
  • Automatically- Self Invoked.

Function Return

Return will stop the function and return a value to it.

function add(x,y){
return x+y;
}
value= add(5,8);

Value would equal 13.
